  • Hoe ga je dan wel te werk in Vim? In Vim typ je commando’s om deze acties uit te voeren. Je gebruikt dus je toetsenbord om aan te geven dat je een woord wil verwijderen of een zin wil kopieren.

Talking with Vim

Veel mensen worden afgeschrikt om Vim te leren, door het feit dat je dan veel commando's moeten vanbuiten leren. Maar eigenlijk valt dat wel mee en men heeft geprobeerd zoveel mogelijk een mnemonisch systeem achter de commando's te steken. Dat is een moeilijk woord dat wil zeggen dat de commando's zeer goed op mensentaal lijken. Voor mij voelt het aan alsof ik gewoon met Vim kan spreken en ik moet dus eigenlijk niets vanbuiten leren.

command

  • d - delete
  • c - change
  • y - yank (copy)
  • v - visually select
  • p - paste
  • i - insert

motion

  • b - begin (of word )
  • e - end (of word)
  • w - (next) word
  • $ - end of line
Hier heb je een aantal basis commando's van vim.
  • Om een actie uit te voeren in vim combineer je een commando met een motion.
  • Deze commando’s en motions zijn zoals je ziet super intuïtief en als je een beetje engels kent is het echt niet moeilijk om te onthouden.
  • Stel dat ik een woord wil wijzigen, dan gebruik ik het commando `c` van change, en ik moet ook meegeven naar waar deze wijzig beweging moet uitgevoerd worden en daarvoor kan ik dus `e` meegeven en dan krijgen we `change end of word`, gemakkelijk om te onthouden he?
  • Stel dat ik alles wil verwijderen van mijn cursor tot en met het begin van dit woord, `db`delete tot begin.
  • Quiz vraagske. Wat als ik van mijn cursor alles wil kopieren tot het volgende woord, Wat typ ik dan?
  • Ik zal dit even demonsteren he. Stel dat ik bijvoorbeeld LekkerBroodje wil kopieren. Hoe zou ik dat doen met de commando's dat we nu kennen?
  • Ik moet navigeren naar het begin van het woord dat ik wil en ik typ `yw` Ik kan dit nu pasten waar ik wil met `p`
  • Stel dat ik nu het woord lekker hier wil verwijderen, hoe kan ik dit doen?

Text Object

  • iw - inside word
  • i{ - inside {
  • it - inside HMTL/XML tag!!
  • tz - (un)til z (on this line)
  • /cyclops - until cyclops
  • a[ - ???
  • at - ???
Nu dat was de echte basis. Daar kan je nog niet zoveel met. Maar nu ga het echte interessante komen!
  • En dat is wanneer we *motion* vervangen door een *text object*.
  • Een text object is een zeer flexibele mannier om een stukje tekst te definiëren.
  • Zo kan ik bijvoorbeeld `iw` typen, dat staat voor /inside word/ en dit definieert het woord onder mijn cursor staat.
  • Dus het maakt niet uit waar in het woord ik sta, ik kan in het midden staat, op het einde, ..
  • Nu kunnen we iets doen met dat woord door dit te combineren met een commando zoals daarnet
  • Dat wil dus zeggen als ik `yiw`typ, dus /yank inside word/, dat ik het woord kopieer onder mijn cursor, zonder dat het uitmaakt waar de cursor in het woord staat. Dus dit woord kan ik nu pasten ergens anders.
  • De kracht hiervan is dat je een text object op zeer veel verchillende mannieren kan definieren

Plugin - Vim Surround

command

  • ds - delete surroundings
  • cs - change surroundings
  • ys - add surroundings (jup, that's a weird one)

Example

  • ds{ - delete surrounding {
  • cs"' - change surrounding " to '
  • cst - change surrounding HTML/XML tag
Vim is uitbreidbaar met plugins, er zijn veel interessante plugins maar ik ga er eentje bespreken die zeker de moeite is voor een developer, namelijk *surround*.
  • Met deze plugin kun je zowat alles doen met betrekking tot de omringende symbolen.
  • Stel dat je bijvoorbeeld dat je double quotes hier wil verwijderen. Dan doe je dit weer met een commando dat zeer veel op spreektaal lijkt. Je doet dit namelijk met `ds”`, waarbij `ds` dus voor /delete surrounding/ staat. Gemakkelijk toch ofniet?
  • We kunnen de symbolen ook wijzigen, stel bijvoorbeeld dat we deze double quotes wil omzetten naar single quotes, dan gebruiken we `cs”’`. We spreken dus weer tegen Vim, we zeggen change surrounding “ to ‘.
  • Dit werkt ook in HTML! Stel dat je deze inhoud vetgedrukt wil. Nu gebruik je `cst*strong*`. Dat is toch fantastisch. We kunnen de tag ook verwijderen met /delete surrounding tag/, dus `dst`.
